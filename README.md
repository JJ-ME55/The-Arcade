# DEEPER

**A deep, tactical, single-player high-score mining game** — a modern, high-class
descendant of the classic Flash game *Motherload*. Dig deep, get rich, don't get stranded.

Built with **Phaser 3 + TypeScript + Vite**. All art and audio are generated
procedurally at runtime (no external asset files, no licensing). Fully playable offline /
locally; works on web **and** mobile (touch joystick).

---

## Run it

```bash
npm install
npm run dev        # http://localhost:5173
```

Build for deployment (static files in `dist/`):

```bash
npm run build      # type-checks then bundles
npm run preview     # serve the production build
```

The whole game is client-side static — `dist/` can be dropped on any static host or
embedded in an iframe.

### Desktop app (Electron)

DEEPER also ships as a standalone desktop application (Windows / macOS / Linux) via Electron.

```bash
npm run electron:dev      # run the desktop app against the live dev server (hot reload)
npm run electron:dir      # quick unpacked build → release/win-unpacked/DEEPER.exe
npm run electron:build    # full installer for the current OS → release/
npm run electron:build:win  # Windows NSIS installer specifically
```

- `electron/main.cjs` is the main process: a portrait game window (600×1000, resizable,
  **F11** toggles fullscreen) that loads the dev server in development and the bundled
  `dist/` when packaged.
- App icon is generated by `node scripts/make-icon.cjs` → `build/icon.png` (pure Node, no
  image deps). electron-builder converts it to platform icons automatically.
- Packaged output (installer / unpacked app) lands in `release/`.

## How it plays

The sacred loop: **dig → fill cargo → fly up → sell → upgrade → go deeper.**

- **Move / fly** — joystick (mobile) or arrow keys / WASD. Hold **up / thrust** to fly;
  gravity always pulls you down. Heavy cargo + weak engine = you can't climb (upgrade!).
- **Dig** — push **down / left / right** into dirt. You can **never dig straight up** —
  this is the core tension; you carve your own tunnels and must plan the climb back.
- **The squeeze** — fuel drains, cargo fills, hull takes damage; all three pull you back
  while exponentially-richer ore pulls you deeper.
- **Surface outpost** — sell ore, refuel, repair, buy upgrades (drill / fuel / cargo /
  hull / engine / radiator / scanner) and consumables (dynamite, teleporter, nanobots…).
- **Score** every run; climb the leaderboard; earn **Cores** to spend on permanent
  perks in the Workshop.

### Modes & live content
- **Free play** — fresh random seed each run.
- **Daily Challenge** — everyone gets the same seeded mine + a rotating **mutator** that
  day (Low Gravity, Fragile Hull, Glass Cannon…). Separate daily board.
- **Seasons** — time-boxed themes (Winterfest, Eastertide, **Midsummer**, Hallowdeep)
  add a signature buried find, a perk, and a battle-pass-lite track of earnable pod skins,
  cores and items. The active season is detected from the real date.
- **Collection** — a museum of every mineral, fossil and artifact you've discovered.
- **Easter eggs** — e.g. a rare **Golden Geode** that cracks into Aurelium.

## Architecture

```
src/
  core/        rng (deterministic spine), types, textures (procedural art),
               save (IndexedDB), state (global App + meta-save)
  config/      ALL content as data: gameplay tuning, biomes, ores, upgrades,
               items, specials, loadouts, seasons, modifiers, metaUpgrades
  world/       world (f(seed,x,y) generation + overrides), tileRenderer
               (chunked + pooled), hazards (boulder dynamics)
  entities/    pod (physics, custom grid collision, digging)
  systems/     stats (derived), run (lifecycle/cargo), score, fx (juice),
               audio (procedural WebAudio), season, runsave (suspend/resume)
  ui/          theme, widgets, hud, touch (joystick), shop (outpost menu)
  scenes/      Boot, Preload, MainMenu, Game, GameOver, HowTo, Leaderboard,
               Collection, Season, Workshop, Settings
```

### Design principles (enforced)
1. **The loop is sacred** — features only land if they make the loop tenser/richer.
2. **Determinism is the spine** — the world is a pure function of `(seed, x, y)`; no
   `Math.random()` in gameplay (only for cosmetic particle jitter). This is what makes
   daily seeds fair and future replay-validation possible.
3. **Mobile is a constraint, not a port** — touch controls were built first.
4. **Every run produces one honest integer score.**
5. **Content is data, not code** — add an ore/biome/upgrade/season/mutator by editing a
   config table; the engine picks it up. No engine changes needed.

### Adding content (examples)
- **New ore:** append to `src/config/ores.ts`.
- **New biome band:** append to `src/config/biomes.ts` (palette + densities).
- **New upgrade tier / item / loadout:** edit the matching `config/` table.
- **New season (e.g. another holiday):** add to `src/config/seasons.ts` with its date
  window, signature find, perk and reward track.
- **New daily mutator:** add to `src/config/modifiers.ts`.

## Saves & leaderboard (local-first)

- **Meta-save** (cores, unlocks, collection, settings, best scores) persists to
  **IndexedDB** (with a localStorage fallback), versioned + migratable.
- **Run save** — an in-progress run is suspended on backgrounding / pausing / visiting the
  outpost, and resumable via **CONTINUE** on the menu (critical for mobile interruptions).
- **Leaderboard** is local, seeded with "ghost" scores so the board feels alive. A cloud
  backend (Supabase Edge Function + HMAC-signed submission + replay validation, per the
  build plan's Phase D) is **scaffolded but disabled** in `src/net/leaderboard.ts` — flip
  `REMOTE_ENABLED` and implement `submitRemote` / `fetchRemote` to go online without any
  UI changes.

## Status vs. the build plan

Implemented: Phase A (core feel + mobile), Phase B (biomes, ores, hazards, boulders,
tools, economy), Phase C (upgrade tree, meta-progression, local + run saves), Phase D
(local leaderboard + daily seeds; cloud + anti-cheat scaffolded), Phase E (seasons,
mutators, daily challenge, Easter eggs), Phase F (procedural art/audio, juice, tutorial,
chunked perf).

Notable future hooks (left intentionally): cloud leaderboard + server replay anti-cheat,
weekly challenge authoring UI, optional account sync, the deep "story / transmissions"
narrative layer.
