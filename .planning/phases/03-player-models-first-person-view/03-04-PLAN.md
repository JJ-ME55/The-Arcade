  Create a FirstPersonWeapon class (ES module) that manages the first-person weapon view.

  **Class API:**
  ```javascript
  export class FirstPersonWeapon {
    constructor(THREE, renderer) { ... }

    // Load FP arms + weapon models, create weapon scene and camera
    async load(armsUrl, weaponUrls) { ... }

    // Switch to weapon by name ('rifle', 'pistol', 'knife')
    switchWeapon(name) { ... }

    // Called each frame -- updates bob, recoil, muzzle flash
    update(dt, state) { ... }

    // Trigger fire effect (recoil + flash)
    fire() { ... }

    // Render the weapon scene (called by main loop after world render)
    render() { ... }

    // Set team color on FP arms
    setTeamColor(hex) { ... }
  }
  ```

  **Weapon scene setup (constructor + load):**
  - Create a separate `THREE.Scene` for the weapon view
  - Create a `THREE.PerspectiveCamera` with same FOV as main camera but near=0.01, far=2.0 (weapon-space only)
  - The weapon camera does NOT move through the world -- it stays at origin. The FP arms + weapon are positioned relative to this camera.

  **Loading (load method):**
  - Load fp_arms.glb with GLTFLoader -> add to weapon scene
  - Load weapon GLBs (rifle.glb, pistol.glb, knife.glb) -> store but only add active weapon to scene
  - Build bone lookup for FP arms (for arm animation later)
  - Create a parent group `this.weaponGroup` that holds FP arms
  - Position weaponGroup relative to weapon camera: roughly (0.15, -0.15, -0.35) -- slightly right, below center, in front of camera. This puts the weapon in the classic FPS bottom-right position.

  **Weapon attachment to Hand.R bone (CRITICAL):**
  Attach each weapon model as a child of the Hand.R bone in the FP arms skeleton. This is the correct approach so the weapon moves with hand animations (recoil, bob, weapon switch).

  Exact attachment code:
  ```javascript
  // After loading fp_arms.glb, find the Hand.R bone:
  let handRBone = null;
  fpArmsScene.traverse((obj) => {
    if (obj.isBone && obj.name === 'Hand.R') {
      handRBone = obj;
    }
  });
  if (!handRBone) {
    // Fallback: try getObjectByName on the skeleton root
    handRBone = fpArmsScene.getObjectByName('Hand.R');
  }
  if (!handRBone) {
    console.error('Hand.R bone not found in fp_arms.glb -- check bone names');
  }
  this.handRBone = handRBone;

  // When switching weapons, attach to Hand.R:
  // Remove previous weapon from Hand.R
  if (this.currentWeaponModel && this.handRBone) {
    this.handRBone.remove(this.currentWeaponModel);
  }
  // Add new weapon as child of Hand.R bone
  this.handRBone.add(weaponModel);
  this.currentWeaponModel = weaponModel;
  ```

  Each weapon needs a local position/rotation offset relative to Hand.R so it looks correct in the hand:
  ```javascript
  const weaponOffsets = {
    rifle:  { pos: [0, 0, -0.15], rot: [0, 0, 0] },
    pistol: { pos: [0, 0, -0.08], rot: [0, 0, 0] },
    knife:  { pos: [0, 0, -0.05], rot: [Math.PI/2, 0, 0] },  // knife blade points forward
  };
  // Apply offset after attaching:
  weaponModel.position.set(...offset.pos);
  weaponModel.rotation.set(...offset.rot);
  ```
  These offsets will likely need empirical tuning. Start with these values and adjust if the weapon looks misaligned.

  **Gun bob (update method):**
  - Only when moving on ground (state.onGround && state.speed > 0.5)
  - Increment `this.bobTime += dt * state.speed * 1.5` (faster bob at higher speed)
  - Bob X (horizontal sway): `sin(bobTime * PI * 2) * 0.008 * speedFactor`
  - Bob Y (vertical bounce): `abs(sin(bobTime * PI * 2)) * 0.006 * speedFactor`
  - speedFactor = `Math.min(state.speed / 4.5, 1.0)` (caps at max run speed)
  - When not moving, smoothly lerp bob offset toward zero (don't snap)
  - Apply bob to weaponGroup position (not individual weapon)

  **Recoil (fire method + update):**
  - On fire: set `this.recoilOffset = 1.0` (instant kick)
  - Each frame: `this.recoilOffset = Math.max(0, this.recoilOffset - recoilDecay * dt)`
  - recoilDecay = 10.0 (spring-back over ~100ms)
  - Apply to weaponGroup: push Z back by `recoilOffset * 0.08`, rotate X up by `recoilOffset * 0.12`
  - For knife: no recoil, instead do a forward slash motion

  **Muzzle flash:**
  - Create a THREE.Sprite with AdditiveBlending:
    ```javascript
    const flashTexture = this._createFlashTexture(); // Procedural canvas gradient
    const flashMaterial = new THREE.SpriteMaterial({
      map: flashTexture,
      blending: THREE.AdditiveBlending,
      color: 0xffaa44,
      transparent: true,
      depthTest: false,
    });
    this.muzzleFlash = new THREE.Sprite(flashMaterial);
    this.muzzleFlash.scale.set(0.12, 0.12, 1);
    this.muzzleFlash.visible = false;
    ```
  - Position at muzzle end of weapon (relative to weapon model)
  - On fire: set visible=true, random rotation, auto-hide after 50ms (3 frames at 60fps)
  - For knife: no muzzle flash

  **Procedural flash texture (_createFlashTexture):**
  ```javascript
  const canvas = document.createElement('canvas');
  canvas.width = 64; canvas.height = 64;
  const ctx = canvas.getContext('2d');
  const gradient = ctx.createRadialGradient(32, 32, 0, 32, 32, 32);
  gradient.addColorStop(0, 'rgba(255, 255, 200, 1)');
  gradient.addColorStop(0.3, 'rgba(255, 180, 50, 0.8)');
  gradient.addColorStop(1, 'rgba(255, 100, 0, 0)');
  ctx.fillStyle = gradient;
  ctx.fillRect(0, 0, 64, 64);
  return new THREE.CanvasTexture(canvas);
  ```

  **Team color (setTeamColor):**
  - Traverse FP arms SkinnedMesh children, clone material if not already cloned, set color

  - `renderer.clearDepth()`
  - `renderer.render(this.weaponScene, this.weaponCamera)`

  3. Modify `_loop()` to use two-pass rendering:
     ```javascript
     // BEFORE (single pass):
     // this.renderer.render(this.scene, this.camera);

     // AFTER (two-pass):
     this.renderer.autoClear = false;
     this.renderer.clear();

     // Pass 1: World scene (map + other players' mannequins)
     this.renderer.render(this.scene, this.camera);

     // Pass 2: First-person weapon (renders on top, no depth conflict with walls)
     if (this.fpWeapon) {
       const speed = Math.hypot(this.vel.x, this.vel.z);
       this.fpWeapon.update(this.dt, {
         speed: speed,
         onGround: this.onGround,
         crouching: this.input.crouch,
       });
       this.fpWeapon.render(this.camera.quaternion);
     }
     ```

  4. Pass the main camera's quaternion to the weapon render so the weapon camera orientation matches.

  5. Add number key bindings for weapon switching (temporary, for testing):
     - Key 1: `fpWeapon.switchWeapon('rifle')`
     - Key 2: `fpWeapon.switchWeapon('pistol')`
     - Key 3: `fpWeapon.switchWeapon('knife')`

  6. Set `this.renderer.autoClear = false` in _initThree or at the start of the render section.

  IMPORTANT: The two-pass technique is:
  1. `renderer.clear()` -- clears color + depth
  2. `renderer.render(worldScene, worldCamera)` -- renders world (writes to depth buffer)
  3. `renderer.clearDepth()` -- clears ONLY depth buffer (keeps color)
  4. `renderer.render(weaponScene, weaponCamera)` -- renders weapon (depth tested only against itself)

  This means the weapon always renders on top of the world, never clipping into walls.
