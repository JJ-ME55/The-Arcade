  ```javascript
    constructor(THREE) { ... }

    // Load the source mannequin GLB (call once)

    // Spawn a player instance with team color, returns instance object

    updateAnimation(instance, dt, state) { ... }

    // Remove instance from scene
    despawn(instance) { ... }
  }
  ```

  **Loading (load method):**
  - Use GLTFLoader (passed via constructor or import) to load mannequin_neutral.glb
  - Store the source scene for cloning
  - Traverse to find and store reference to the SkinnedMesh and Skeleton for validation
  - Log bone names found to console for debugging

  **Spawning (spawn method):**
  - Use `SkeletonUtils.clone(this.sourceScene)` to create independent instance
  - Build bone lookup map: `{ 'Root': bone, 'Thigh.L': bone, ... }`
  - Clone materials and set team color: `child.material = child.material.clone(); child.material.color.setHex(teamColor);`
  - Set `frustumCulled = false` on SkinnedMesh (animated meshes can move outside their original bounding box)
  - Add to scene at given position
  - Return instance object containing: `{ model, bones, scene }`

  **Team colors:**
  - Red team: `0xcc2200` (solid matte red)
  - Blue team: `0x2244cc` (solid matte blue)

  **Procedural animation (updateAnimation method):**

  The `state` parameter contains: `{ velocity: Vector3, onGround: boolean, crouching: boolean, time: number, shooting: boolean, reloading: boolean }`

  Animation state machine:
  1. **Idle** (onGround, speed < 0.5): Reset all bone rotations to 0. Subtle breathing: `Chest.rotation.x = Math.sin(time * 1.5) * 0.01`
  2. **Run** (onGround, speed >= 0.5):
     - Frequency = speed * 1.2 (cycles/sec), phase `t = time * freq * PI * 2`
     - Amplitude scales with speed: `amp = Math.min(speed / 4.5, 1.0)`
     - Thigh.L.rotation.x = sin(t) * 0.5 * amp (forward/back swing)
     - Thigh.R.rotation.x = sin(t + PI) * 0.5 * amp (opposite phase)
     - Shin.L.rotation.x = -abs(sin(t)) * 0.4 * amp (knee bend, always negative)
     - Shin.R.rotation.x = -abs(sin(t + PI)) * 0.4 * amp
     - UpperArm.L.rotation.x = sin(t + PI) * 0.35 * amp (opposite to legs)
     - UpperArm.R.rotation.x = sin(t) * 0.35 * amp
     - ForeArm.L.rotation.x = -0.2 - abs(sin(t + PI)) * 0.2 * amp (elbow slightly bent)
     - ForeArm.R.rotation.x = -0.2 - abs(sin(t)) * 0.2 * amp
     - Root.position.y += abs(sin(t)) * 0.015 * amp (body bob)
     - Chest.rotation.y = sin(t) * 0.04 * amp (torso twist)

  3. **Strafe** (onGround, lateral velocity dominant):
     - Detect strafe: `abs(lateralVel) > abs(forwardVel) * 0.5`
     - Lean torso in strafe direction: `Chest.rotation.z = -sign(lateralVel) * 0.08 * amp`
     - Legs do a side-step shuffle: smaller stride, more lateral
     - Reduce arm swing amplitude by 50%

  4. **Jump** (!onGround):
     - Tuck legs slightly: Thigh rotation -0.2, Shin rotation -0.3
     - Arms slightly out: UpperArm rotation.z = +-0.2 (spread)
     - If rising (vel.y > 0): more tuck
     - If falling (vel.y < 0): legs extend slightly

  5. **Crouch** (crouching):
     - Root.position.y -= 0.3 (lower body center)
     - Thighs bend more: rotation.x -= 0.4
     - Shins bend: rotation.x -= 0.3
     - When crouching AND moving, apply run animation at 60% amplitude

  6. **Shooting** (state.shooting):
     - Quick recoil kick on UpperArm.R: rotation.x -= 0.1 (pushes arm back)
     - Spring back over 0.1s

  7. **Reload** (state.reloading):
     - Left arm reaches across: UpperArm.L.rotation.z = -0.3, ForeArm.L.rotation.x = -0.8
     - Cycle over 2 seconds, return to normal

  **IMPORTANT bone rotation axis notes:**
  After Blender Z-up to GLTF Y-up conversion, bone axes may differ from Blender space. The research doc warns this needs empirical testing. Start with the rotation axes as described (rotation.x for forward/back swing since bones point along Y). If bones rotate in unexpected directions after loading, the executor should test with SkeletonHelper + AxesHelper and adjust rotation axes accordingly. Include a debug mode that adds SkeletonHelper to the model.

  **Transform (setTransform method):**

  - Set `instance.model.rotation.y = yaw` (face the direction the player is looking)
