<task type="auto">
  <name>Task 1: Create Blender mannequin with armature and geometric body parts</name>
  <files>visual/blender/create_mannequin.py</files>
  <action>
  Create a Blender Python script that builds a humanoid mannequin from scratch when run in Blender background mode. The script must:

  1. **Clear the scene** -- delete all existing objects (the script creates its own scene from scratch).

  2. **Create armature** with 22 bones in this hierarchy:
     ```
     Root (hips, ~0.95m height)
       +-- Spine -> Chest -> Neck -> Head
       +-- Shoulder.L -> UpperArm.L -> ForeArm.L -> Hand.L
       +-- Shoulder.R -> UpperArm.R -> ForeArm.R -> Hand.R
       +-- Thigh.L -> Shin.L -> Foot.L
       +-- Thigh.R -> Shin.R -> Foot.R
     ```
     Use the bone positions from the research doc (Root at Z=0.95, Head top at Z=1.74, etc). Total mannequin height = 1.8m to match the player capsule collision shape.

  3. **Create mesh body parts** using Blender primitives:
     - **Head**: UV sphere, radius ~0.12m, at Z=1.62
     - **Neck**: Cylinder, radius ~0.04m, depth ~0.10m, at Z=1.45
     - **Chest/Torso**: Cylinder, radius ~0.14m (slightly wider), depth ~0.35m, at Z=1.22 (covers spine through chest)
     - **Hips/Pelvis**: Cylinder, radius ~0.13m, depth ~0.15m, at Z=0.97
     - **Shoulders**: Small spheres, radius ~0.06m, at shoulder positions (x=+-0.18, z=1.38)
     - **Upper Arms**: Cylinders, radius ~0.05m, depth ~0.28m, oriented vertically from shoulder down
     - **Forearms**: Cylinders, radius ~0.04m, depth ~0.26m, continuing from upper arms
     - **Hands**: Small cubes/spheres, ~0.06m, at wrist positions
     - **Thighs**: Cylinders, radius ~0.06m, depth ~0.45m, from hips to knees
     - **Shins**: Cylinders, radius ~0.05m, depth ~0.42m, from knees to ankles
     - **Feet**: Scaled cubes, ~0.08 x 0.12 x 0.04m, at ground level

     All body parts use Blender Z-up convention. Positions must be accurate to within 0.02m.

  4. **Assign vertex groups** -- For each mesh body part, create a vertex group whose name exactly matches the bone that controls it. Add ALL vertices of that mesh to the group with weight 1.0. This is critical for the GLB export to contain valid skinning data.

  5. **Parent to armature** -- Set each mesh's parent to the armature object. Add an Armature modifier to each mesh with `use_vertex_groups=True` and `use_bone_envelopes=False`.

  6. **Apply transforms** -- Before parenting, apply scale/rotation on all mesh objects so they're clean (`bpy.ops.object.transform_apply(location=False, rotation=True, scale=True)`). Do NOT apply location (meshes need their world position for the rest pose). Do NOT apply the Armature modifier -- the GLTF exporter handles it.

  7. **Material** -- Create one neutral grey MeshStandardMaterial-equivalent (Principled BSDF, Base Color = (0.6, 0.6, 0.6, 1.0), Roughness = 0.9, Metallic = 0.0). Assign it to ALL body part meshes. Team color will be swapped at runtime in Three.js.

  8. **Save** -- Save the Blender file to `visual/blender/mannequin.blend`.

  9. **Logging** -- At the end of the script, print summary counts to stdout for verification:
     - Total bone count in the armature (expected: 22)
     - Total mesh object count (list each mesh name)
     - Total vertex group count across all meshes
     - Confirmation that Armature modifier was added to each mesh
     Format: `print(f"SUMMARY: {bone_count} bones, {mesh_count} meshes, {vgroup_count} vertex groups")`

  Proportions guide (7.5 head-heights, head = 0.24m):
  - Head top: 1.8m (but model stops at ~1.74 since skull is sphere)
  - Shoulders: 1.38m
  - Elbows: 1.10m
  - Wrists: 0.84m
  - Hips: 0.95m
  - Knees: 0.50m
  - Ankles: 0.08m
  - Ground: 0.0m

  IMPORTANT: Bone orientations in Blender -- bones point from head to tail along their local Y axis. For arm bones, orient them pointing downward (head at shoulder, tail at elbow). For leg bones, head at hip, tail at knee. This is critical for correct rotation axes after GLTF Y-up conversion.

  Run command:
  ```
  "/c/Program Files/Blender Foundation/Blender 5.0/blender.exe" --background --python visual/blender/create_mannequin.py
  ```
  </action>

  3. Script output logs confirm: 22 bones created, all body part meshes created, all vertex groups assigned, armature modifier added to each mesh

<task type="auto">
  <name>Task 2: Export mannequin to GLB with skinning data</name>
  <files>visual/blender/export_mannequin.py, visual/mannequin_neutral.glb</files>
  <action>
  Create an export script that opens mannequin.blend and exports to GLB:

  ```python
  import bpy

  # Select all objects in scene
  bpy.ops.object.select_all(action='SELECT')

  bpy.ops.export_scene.gltf(
      filepath='visual/mannequin_neutral.glb',
      export_format='GLB',
      export_skins=True,           # CRITICAL: preserves armature/skinning
      export_yup=True,             # Convert Blender Z-up to glTF Y-up
      export_apply=True,           # Apply modifiers (except armature, handled by exporter)
      export_extras=True,          # Custom properties
      export_animations=False,     # No baked animations -- all procedural
      export_cameras=False,
      export_lights=False,
      use_selection=False,         # Export all objects
  )
  print("Export complete: visual/mannequin_neutral.glb")
  ```

  Run command:
  ```
  "/c/Program Files/Blender Foundation/Blender 5.0/blender.exe" --background "visual/blender/mannequin.blend" --python visual/blender/export_mannequin.py
  ```

  IMPORTANT: The `export_skins=True` parameter may have a different name in Blender 5.0. If the export fails with a parameter error, check available parameters by running:
  ```python
  import bpy
  import inspect
  sig = inspect.signature(bpy.ops.export_scene.gltf)
  for p in sorted(sig.parameters): print(p)
  ```
  And find the equivalent skin-export parameter. The research doc flags this as MEDIUM confidence for Blender 5.0.

  Also verify `filepath` uses forward slashes and is relative to the Blender working directory (CWD when running background mode). If Blender CWD is the project root, use `visual/mannequin_neutral.glb`. If not, use an absolute path.
