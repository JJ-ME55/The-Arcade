# 04.1-03 SUMMARY — Soldier integration, animation, team color

**Status:** Core complete (combat/crouch/jump overlays implemented, final sign-off deferred to 04.1-06)
**Date:** 2026-05-22

## What was built
The realistic soldier (`soldier.glb`) is now the universal third-person player avatar, animated and team-colored, with all prototype scaffolding removed.

## Done
- **Load swap + scaffolding removal (Task 1):** `visual/main.js` loads `soldier.glb`; deleted `_loadSoldierProto()`, `soldierMixer`, and `soldier_proto.glb`. No startup errors.
- **Team color:** `spawn()` applies `material.color` tint + low emissive accent — RED/BLUE clearly distinguishable in-engine (user-confirmed).
- **Natural locomotion (major deviation — see below):** idle/walk/run via baked Mixamo mocap blended by speed. User reaction: "this looks fantastic."

## Deviations (justified, user-approved)
1. **Bone-name dot aliasing.** three.js GLTFLoader strips `.` from node names (`UpperArm.L`→`UpperArmL`), so *every* dotted bone lookup in the animation code silently returned undefined — the model never animated. Added a dotted→dedotted alias map in `spawn()`. (Root cause of the long "nothing moves" debugging.)
2. **Rest-relative procedural animation.** Mixamo bones rest at non-identity rotations; the old code zeroed rotations (contorting the model). Refactored `_resetBones` to restore captured bind quaternions and animations to apply local-axis deltas via `_rot`. (Retained as the no-clip fallback.)
3. **Locomotion switched procedural → baked mocap clips.** The procedural sine-swing read as a stiff march. Per research (CS uses baked/mocap clips + blending; procedural only for overlays), switched idle/walk/run to the Mixamo clips bundled in the source GLB, blended by speed via `AnimationMixer`. Combat (shoot/reload/knife) + crouch/jump are procedural overlays layered on top. See [[opponent-model-direction]].
4. **No scale bake in the rig export.** Baking scale rescaled the skeleton but not the cm-authored clip translations, flinging the model off-screen. Fixed by keeping native node scale (0.01) and baking rotation only. Soldier renders ~1.73m.
