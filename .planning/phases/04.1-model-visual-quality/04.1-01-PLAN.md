The chosen approach determines the rename script and whether vertices need re-weighting. Recommend trying Option A first (delete + reweight) and falling back to B only if the torso pinches.
  </context>
  <options>
    <option id="delete-reweight">
      <name>Delete Spine2, re-weight its vertices to Chest (Spine1)</name>
      <pros>Cleanest 19-bone skeleton exactly matching the custom rig; no orphan bones; ragdoll/hitbox lookups unambiguous</pros>
      <cons>Requires vertex re-weighting in Blender; risk of torso pinch artifact at the merge seam</cons>
    </option>
    <option id="rename-ignored">
      <name>Keep Spine2, rename to "Chest2" (a name the procedural code never references)</name>
      <pros>Zero re-weighting; no deformation risk; procedural code ignores unknown bone names</pros>
      <cons>Extra bone in skeleton; must ensure no two bones map to a name the code reads (avoid duplicate "Chest")</cons>
    </option>
  </options>

  <action>
Using the Blender MCP tools (execute_blender_code), write and run a script saved to `visual/blender/migrate_soldier_rig.py` that performs the full Option A migration. The user will have downloaded the Mixamo soldier FBX (see user_setup); ask for / locate its path. If the FBX is not yet present, proceed against the existing `visual/soldier_proto.glb` (same Mixamo skeleton) so dev can continue — the licensed FBX is a late same-skeleton swap.

Steps the script must perform:
1. Import the Mixamo FBX. CRITICAL (RESEARCH.md Pitfall 3): Mixamo FBX is centimeter-scale (0.01). After import run `bpy.ops.object.transform_apply(scale=True)` and verify the armature/mesh bounding-box height is ~1.8m (matching the player capsule and the existing mannequin_neutral.glb). Fix scale if wrong.
2. Rename bones: strip the `mixamorig:` prefix, then apply the RENAME_MAP from RESEARCH.md (Hips→Root, Spine→Spine, Spine1→Chest, Neck→Neck, Head→Head, Left/RightShoulder→Shoulder.L/.R, Left/RightArm→UpperArm.L/.R, Left/RightForeArm→ForeArm.L/.R, Left/RightHand→Hand.L/.R, Left/RightUpLeg→Thigh.L/.R, Left/RightLeg→Shin.L/.R, Left/RightFoot→Foot.L/.R). Bone renames must be done in Edit/Object mode on `armature.edit_bones` or `armature.bones`.
3. Handle Spine2 per the decision checkpoint above (delete+reweight, or rename to Chest2). Ensure NO bone is named "Chest" twice.
4. Pose the armature so the arms hang straight down matching `visual/mannequin_neutral.glb`'s neutral rest pose (Mixamo imports in T or A pose). Then in Pose Mode run Apply → Apply Pose as Rest Pose so all bone rotations bake to zero (RESEARCH.md "The Mixamo Rest-Pose Problem").
5. Export GLB to `visual/soldier.glb` with export_skins=True, export_yup=True, and NO animations (use `bpy.ops.export_scene.gltf(filepath=..., export_format='GLB', export_skins=True, export_yup=True, export_animations=False)`).
6. Take a get_viewport_screenshot for the SUMMARY showing the renamed-bone armature in neutral pose.

Do NOT compress textures here — that is a separate concern handled during game integration if load times are an issue. Keep this plan to producing a correct, correctly-named, correctly-posed GLB.
