# 04.1-01 SUMMARY — Soldier rig migration

**Status:** Complete
**Date:** 2026-05-22

## What was built
`visual/soldier.glb` — a realistic human soldier (the Mixamo "Vanguard" mesh) re-rigged so the existing procedural animation system in `src/player-model.js` can drive it by bone name. Reproducible via `visual/blender/migrate_soldier_rig.py` (Blender 5.0 headless).

## Verification (met)
- All 19 custom bone names present and unique: Root, Spine, Chest, Neck, Head, Shoulder.L/.R, UpperArm.L/.R, ForeArm.L/.R, Hand.L/.R, Thigh.L/.R, Shin.L/.R, Foot.L/.R (exactly one "Chest").
- Height normalized to 1.800 m (matches player capsule / old mannequin).
- Mesh intact (workbench render confirms a clean soldier, natural A-pose, no deformation artifacts).
- Script runs end-to-end: `blender --background --python visual/blender/migrate_soldier_rig.py -- <src> <out.glb> <preview.png> <arm_deg>`.

## Decisions & deviations
- **Spine2 → "Chest2" (rename-ignored)**, per the plan's documented fallback. Zero re-weighting risk; `player-model.js` never references "Chest2" so it's inert. (Can revisit delete-reweight on the final licensed model if the torso ever pinches.)
- **Source = `soldier_proto.glb` stand-in** (same Mixamo skeleton as the licensed Vanguard). The licensed Mixamo FBX is a late same-skeleton swap through this same script (the script already branches on `.fbx` vs `.glb`).
- **Rest-pose bake DISABLED.** "Apply Pose as Rest Pose" on this skinned rig in *headless* Blender corrupted the skin bind (exploded mesh — confirmed via render). Rotation-apply on a skinned armature has the same effect. So the script applies **scale only** and exports the natural bind pose (a clean A-pose). The arms-down neutral stance + per-bone axis correction are deferred to `src/player-model.js` (plan 03), iterated against the live game renderer (reliable) instead of blind Blender. This is consistent with plan 03 Task 2's intent.
