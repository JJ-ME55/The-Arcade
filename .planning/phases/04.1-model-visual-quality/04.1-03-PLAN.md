  <action>
The migrated soldier may have subtly different bone local axes than the original mannequin (RESEARCH Pitfall 1 / "Caveat"). Verify each animation state and apply minimal per-bone axis fixes ONLY where wrong:

1. Enable debug mode: set `this.debugMode = true` temporarily (or wire a key) so SkeletonHelper is visible on spawned instances. The red/blue/green test soldiers from `_spawnTestMannequins()` provide live subjects.
2. Drive each state and observe direction of motion against the expected behavior documented in the `_animateRun/_animateStrafe/_animateJump/_animateCrouch/_animateShooting/_animateReload/_animateKnife` JSDoc:
   - Run: thighs should swing forward/back (along the body's sagittal plane), shins bend, arms counter-swing. If a limb swings sideways instead, that bone's axis is wrong.
   - Strafe lean (Spine rotation.z), jump tuck (Thigh +x / Shin -x), crouch lower (Root.position.y) + bend, shoot recoil (UpperArm.R -x), reload (UpperArm.L y/z reach), knife (UpperArm.R x/y slash).
3. For any bone animating in the wrong plane, apply the minimal fix the research prescribes: swap the rotation axis (e.g. `.rotation.x` → `.rotation.z`) or flip the sign (`+` → `-`) for that specific bone in that specific `_animate*` function. Do NOT rewrite the animation math — only correct axis/sign per bone. Document each change made.
4. Team color (RESEARCH "Team Color Approach", recommended order): the existing `spawn()` already does `child.material.color.setHex(teamColor)`. On the soldier's PBR texture this tints but may be subtle. Add a low-intensity emissive accent in `spawn()`:
   `child.material.emissive = new THREE.Color(teamColor); child.material.emissiveIntensity = 0.15;`
   Keep the existing team color hexes (red 0xcc2200, blue 0x2244cc as used by callers). If tint+emissive still reads ambiguously at distance, bump emissiveIntensity toward 0.3 (do not exceed glowing). This is the cheap path; a Blender insignia is out of scope for this plan.
5. Turn debugMode back off (default false) before finishing.
