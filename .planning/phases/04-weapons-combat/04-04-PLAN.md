<objective>
Build the visual combat feedback system with object-pooled particles, decals, tracers, and hit effects.

Purpose: Visual feedback is what makes combat FEEL impactful. Blood sprays, sparks flying off walls, bullet holes accumulating, tracers streaking through the air — these transform hitscan math into visceral combat. Without this, shots feel like clicking into void. Object pooling is mandatory to maintain 60fps during firefights.

Output: Single JavaScript module `combat-feedback.js` with all visual combat effects, ready for integration. Uses Three.js directly (not an engine module — this is a visual system).
</objective>

<task type="auto">
  <name>Task 1: Blood Particles and Spark Effects (src/combat-feedback.js - Part 1)</name>
  <files>src/combat-feedback.js</files>
  <action>
Create the combat feedback module. This is a JavaScript module (not TypeScript) because it uses Three.js directly for visual effects.

**CombatFeedback class constructor(THREE, scene):**
- Stores THREE reference and scene
- Initializes all particle pools on construction

**Blood Particle Pool (300 sprites):**
- Pre-allocate 300 THREE.Sprite objects with blood splatter texture
- Blood texture: procedural CanvasTexture (64x64) — irregular dark red (0x8b0000) splatter shape using overlapping circles at random angles
- SpriteMaterial: NormalBlending, transparent, depthWrite=false
- Initial scale: 0.1-0.15 (randomized per spawn)
- All sprites added to scene, set visible=false initially

`onPlayerHit(position, direction, isHeadshot):`
- Spawn 5-8 blood particles from the hit position
- Velocity: opposite of shot direction + random spread (0.5 cone) + upward bias
- Speed: 2-4 m/s randomized
- Gravity: -9.8 m/s^2
- Lifetime: 1.0-1.5s with fade-out starting at 50% lifetime
- If isHeadshot: spawn 10-12 particles (more dramatic), larger scale (0.15-0.2)

**Helmet Spark (headshot feedback):**
- On headshot: additionally spawn a single bright flash at hit position
- Use a separate small pool of 5 spark sprites (yellow-white, AdditiveBlending)
- Flash duration: 80ms, scale pulses 0.1 -> 0.25 -> 0 over that time
- This is the visual component of the "dink" — audio handled in Phase 6

**Environment Spark Pool (200 sprites):**
- Pre-allocate 200 orange/yellow spark sprites
- SpriteMaterial: AdditiveBlending, transparent, depthWrite=false
- Smaller scale: 0.03-0.06

`onEnvironmentHit(position, normal):`
- Spawn 3-5 spark sprites from surface position
- Velocity: reflect shot direction off surface normal + random spread
- Speed: 3-6 m/s
- Gravity: -15 m/s^2 (sparks fall faster than blood)
- Lifetime: 0.3-0.5s with rapid fade

**update(dt) method:**
- Iterate all active particles (blood + sparks + helmet sparks)
- Apply velocity, gravity, fade
- Return expired particles to their respective pools
- Use reverse iteration + splice for efficient removal

**Pool management:**
- Each pool tracks: `pool[]` (available) and `active[]` (in use)
- `_acquireFromPool(pool, active)` — pop from pool, push to active, return sprite
- `_releaseToPool(pool, active, index)` — splice from active, push to pool, hide sprite
- When pool exhausted: fail silently (return without spawning)

<task type="auto">
  <name>Task 2: Bullet Decals, Tracers, and Enemy Muzzle Flash (src/combat-feedback.js - Part 2)</name>
  <files>src/combat-feedback.js</files>
  <action>
Add to the existing CombatFeedback class (extend the file from Task 1):

**Bullet Decal System:**
- Use THREE.DecalGeometry for bullet holes on environment surfaces
- Pre-create a bullet hole texture: 32x32 CanvasTexture — dark circle with radial cracks pattern
- DecalMaterial: MeshBasicMaterial, transparent, polygonOffset (to prevent z-fighting), opacity 0.8

`addBulletDecal(position, normal, targetMesh):`
- Create DecalGeometry projected onto targetMesh at position with normal orientation
- Size: 0.08 x 0.08 x 0.05 (small bullet hole)
- Add to scene
- Track in `decals[]` array with creation time
- Limit: max 100 active decals. When exceeded, remove oldest (FIFO).
- NOTE: DecalGeometry requires the target mesh and intersection data. If targetMesh is not available (no world mesh at hit point), skip decal and only show sparks.

**Bullet Tracer System:**
- Pool of 20 tracer line segments (THREE.Line with BufferGeometry of 2 points)
- Material: LineBasicMaterial, color 0xffdd44 (yellow-orange), transparent, linewidth 1
- Tracers show every Nth bullet (configured externally, default N=4)

`spawnTracer(startPosition, endPosition):`
- Acquire tracer from pool
- Set geometry positions: start = muzzle position, end = hit position (or max range point)
- Visible = true
- Tracer lifetime: 0.15s (very brief streak)
- During lifetime: tracer "slides" — start position moves toward end position over time (trail effect)

**Enemy Muzzle Flash System:**
- Pool of 8 muzzle flash sprites (for rendering other players' weapon fire)
- Same procedural radial gradient texture as FirstPersonWeapon muzzle flash
- AdditiveBlending, scale 0.2-0.3

`showEnemyMuzzleFlash(position):`
- Acquire flash from pool
- Position at given world position
- Duration: 50ms, same random rotation as FP muzzle flash
- Visible at distance (useful for locating enemy fire)

**update(dt) method (extend existing):**
- Also update tracers: slide start toward end, fade, return to pool when expired
- Also update enemy muzzle flashes: timer-based, return to pool when expired
- Also handle decal cleanup: remove decals older than 60 seconds to prevent memory growth

**Full API summary for CombatFeedback:**
```javascript
class CombatFeedback {
  constructor(THREE, scene)
  onPlayerHit(position, direction, isHeadshot)   // Blood + helmet spark
  onEnvironmentHit(position, normal)              // Sparks
  addBulletDecal(position, normal, targetMesh)    // Bullet hole
  spawnTracer(startPosition, endPosition)         // Bullet tracer line
  showEnemyMuzzleFlash(position)                  // Third-person muzzle flash
  update(dt)                                       // Animate all active effects
  dispose()                                        // Cleanup all resources
}
```

Export the CombatFeedback class as default export.
