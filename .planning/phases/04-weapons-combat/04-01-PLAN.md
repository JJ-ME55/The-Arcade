  <name>Task 1: Weapon State Machine and Definitions (src/engine/weapons.ts)</name>
  <files>src/engine/weapons.ts</files>
  <action>
Create the weapon system engine module with:

**WeaponType enum:** `AK47`, `M4A1`, `PISTOL`, `KNIFE`

**WeaponConfig per weapon (from CONTEXT.md decisions):**
- AK-47: 600 RPM (0.1s cycle), 36 base damage, 30/90 mag/reserve, 2.5s reload, 0.7s draw, 215 u/s speed
- M4A1: 667 RPM (0.09s cycle), 32 base damage, 30/90 mag/reserve, 3.1s reload, 0.7s draw, 221 u/s speed
- Pistol: ~400 RPM (0.15s cycle), ~25 base damage, 20/120 mag/reserve, 2.2s reload, 0.5s draw, 240 u/s speed
- Knife: 0.4s swing (left), 0.8s stab (right), ~40 left-click damage, instant kill right-click backstab, 0.25s draw, 250 u/s speed

**WeaponState enum:** `IDLE`, `FIRING`, `RELOADING`, `DRAWING`, `KNIFE_ATTACK`

**WeaponSystem class:**
- `currentWeapon: WeaponType` - tracks active weapon
- `state: WeaponState` with `stateTime: number` tracking
- `magazine: number` and `reserve: number` per weapon (stored in a Map)
- `shotsFired: number` — spray pattern index (increments on fire, resets after 0.45s no fire)
- `lastShotTime: number` — for spray reset detection

**State machine update(dt):**
- DRAWING: blocks fire/reload until drawTime elapsed, then transitions to IDLE
- RELOADING: commits once started (CS:S behavior), cannot cancel. Ammo refills at 50% through animation (D8). Transitions to IDLE when done.
- FIRING: cooldown based on fire rate, then transitions to IDLE
- KNIFE_ATTACK: blocks all other actions during attack animation
- IDLE: accepts fire/reload/switch inputs

**Methods:**
- `canFire(): boolean` — state is IDLE AND fire rate cooldown elapsed AND magazine > 0
- `canReload(): boolean` — state is IDLE AND magazine < max AND reserve > 0 AND weapon is not knife
- `canSwitch(): boolean` — state is IDLE or FIRING (not during reload or draw)
- `fire(): FireResult | null` — decrements mag, increments shotsFired, updates lastShotTime, returns { weaponType, shotIndex, damage } or null if can't fire. Auto-reload if mag hits 0.
- `startReload(): boolean` — starts reload if canReload()
- `switchWeapon(type: WeaponType): boolean` — starts DRAWING state, resets shotsFired
- `knifeAttack(isRightClick: boolean): KnifeResult | null` — left: 40 damage, right: backstab flag
- `getMovementSpeed(): number` — returns movement speed for current weapon (HU/s values above)
- `getWeaponConfig(): WeaponConfig` — returns config for current weapon
- `resetAll()` — full ammo reset (for round start)

**Export types:** WeaponType, WeaponState, WeaponConfig, WeaponSystem, FireResult, KnifeResult

Use the Vec3 type pattern from movement.ts. Do NOT import THREE.js — this is a pure engine module.

<task type="auto">
  <name>Task 2: Fixed Spray Patterns and Accuracy Model (src/engine/recoil-patterns.ts)</name>
  <files>src/engine/recoil-patterns.ts</files>
  <action>
Create the recoil and accuracy engine module with:

**Fixed 30-shot spray patterns (CS:S faithful):**

AK-47 pattern (harder recoil — climbs up, pulls LEFT after ~5-6 shots):
- Shots 1-4: Strong upward climb, zero horizontal
- Shots 5-10: Continued upward, begins pulling left
- Shots 11-15: Moderate upward, strong left pull
- Shots 16-20: Reduced upward, oscillates right
- Shots 21-25: Minimal upward, pulls right
- Shots 26-30: Slight downward, returns center-left

Each entry is `{ x: number, y: number }` where x is horizontal (negative=left, positive=right) and y is vertical (positive=up). Values are in degrees of angular offset from crosshair center. Scale so the full pattern spans roughly 6-8 degrees vertical and 3-4 degrees horizontal. These are CUMULATIVE offsets from the resting crosshair position, not per-shot deltas.

Wait — actually store them as PER-SHOT DELTA angles (how much this shot offsets from the previous). This is more natural for the recoil system. Shot 1 delta might be (0, 0.6 deg up). Shot 2 delta (0, 0.8 deg up). Shot 6 delta (-0.3 deg left, 0.4 deg up). Etc.

M4A1 pattern (easier recoil — climbs up, drifts RIGHT, more controllable):
- Same structure but smaller values (~70% of AK vertical, horizontal drifts right instead of left)
- More predictable — less lateral variance

**Pistol:** No fixed pattern. Each shot applies a small random spread + recovery time.

**getRecoilAngle(weaponType, shotIndex) -> { x: number, y: number }:**
- Returns the per-shot delta angle from the pattern array
- For shotIndex beyond pattern length (30), wraps around to create repeating pattern
- For pistol, returns small fixed upward kick with minor random horizontal

**AccuracyModel class:**
- `accuracy: number` (0.0 = wildly inaccurate, 1.0 = perfect)
- `update(dt, speed, onGround, crouching, timeSinceLastShot):`
  - Airborne: accuracy = 0.0 (rifles completely inaccurate, pistol very inaccurate)
  - Moving (speed > threshold): rifles accuracy ~0.1, pistol ~0.4
  - Standing still: accuracy = 1.0 (perfect first-shot accuracy)
  - Counter-strafe: INSTANT accuracy restore when speed drops below threshold (core CS:S mechanic)
  - Crouching bonus: tighter spread (multiply spread by 0.7 when crouched)
  - Spray penalty: accuracy degrades with sustained fire, recovers over 0.45s
- `getSpreadAngle(accuracy) -> { x: number, y: number }:`
  - Random angle within a cone whose radius is `(1 - accuracy) * maxSpread`
  - maxSpread: ~5 degrees for rifles, ~3 degrees for pistol
  - Returns random offset to add ON TOP of the spray pattern angle

**getFinalShotAngle(weaponType, shotIndex, accuracy) -> { x: number, y: number }:**
- Combines: pattern delta + random spread
- This is the total angular offset for this particular shot

Use plain math (Math.sin, Math.cos, Math.random). No THREE.js imports. Export all pattern arrays as const for potential debug visualization.
