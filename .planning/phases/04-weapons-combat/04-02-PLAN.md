<objective>
Build the hitbox geometry system and hitscan raycasting engine for combat hit detection.

Purpose: Every shot in the game tests a ray against animated primitive hitboxes. This must be fast (custom ray-primitive math, not Three.js Raycaster), accurate (hitboxes track skeleton bones), and slightly generous (head hitbox 15% larger, CS:S feel). Arm penetration ensures arms don't act as shields.

Output: Two TypeScript engine modules: `hitboxes.ts` (ray-primitive intersection math + hitbox definitions) and `combat.ts` (hitscan testing, hitbox set management, hit results).

**HitboxZone type:**
```typescript
type HitboxZone = 'head' | 'chest' | 'stomach' | 'arm_l' | 'arm_r' | 'leg_l' | 'leg_r';
```

**Armor protection flags:**
- head: protected only if hasHelmet
- chest: protected if hasArmor
- stomach: protected if hasArmor
- arms: protected if hasArmor
- legs: NOT protected by armor

**Hitbox interface:**
```typescript
interface Hitbox {
  zone: HitboxZone;
  shape: 'sphere' | 'capsule' | 'box';
  multiplier: number;
  armorProtected: boolean;
  // Sphere: center + radius
  center: Vec3;
  radius?: number;
  // Capsule: two endpoints + radius
  endA?: Vec3;
  endB?: Vec3;
  // Box: center + halfExtents + rotation (as 3x3 matrix or euler)
  halfExtents?: Vec3;
  rotation?: [number, number, number]; // euler angles
  // Bone name this hitbox follows
  boneName: string;
}
```

**HitboxSet:** Array of 7 Hitbox objects (one per zone) for a single player.

**raySphereIntersect(rayOrigin: Vec3, rayDir: Vec3, center: Vec3, radius: number): number | null**
- Standard ray-sphere intersection using quadratic formula
- Returns distance along ray to closest intersection point, or null if no hit
- MUST reject negative t values (intersection behind ray origin)
- Use EPSILON = 0.001 to avoid self-intersection artifacts

**rayCapsuleIntersect(rayOrigin: Vec3, rayDir: Vec3, endA: Vec3, endB: Vec3, radius: number): number | null**
- Capsule = cylinder with hemisphere caps
- Test infinite cylinder first (ray vs line segment distance)
- Clamp to segment [endA, endB]
- If outside segment, test hemisphere caps at endA and endB
- Return closest positive intersection, or null
- Reference: Inigo Quilez intersectors (https://iquilezles.org/articles/intersectors/)

**rayBoxIntersect(rayOrigin: Vec3, rayDir: Vec3, center: Vec3, halfExtents: Vec3, rotation?: [number, number, number]): number | null**
- Oriented bounding box intersection (slab method)
- If rotation provided, transform ray into box local space first
- Standard AABB slab test: compute tmin/tmax for each axis, take max(tmin), min(tmax)
- Return tmin if tmin <= tmax and tmin > EPSILON, else null

**Default hitbox template (from mannequin bone structure):**
Create a function `createDefaultHitboxTemplate(): Hitbox[]` that returns the 7 hitbox definitions with approximate sizes based on the mannequin model (from player-model.js bones: Root, Spine, Chest, Head, UpperArm.L/R, ForeArm.L/R, Hand.L/R, Thigh.L/R, Shin.L/R):
- Head: sphere, radius ~0.14 (15% LARGER than visual head ~0.12), follows Head bone
- Chest: box, ~0.3w x 0.25h x 0.2d, follows Chest bone
- Stomach: box, ~0.28w x 0.2h x 0.18d, follows Spine bone
- Arms (L/R): capsule, radius ~0.06, from UpperArm to Hand bone
- Legs (L/R): capsule, radius ~0.08, from Thigh to foot (Shin end)

Export all types, functions, and the ZONE_MULTIPLIERS constant.

Create the combat engine module that orchestrates hitscan testing against players.

**Import from hitboxes.ts:** All ray-primitive functions, Hitbox, HitboxZone, HitboxSet, ZONE_MULTIPLIERS

**HitResult type:**
```typescript
interface HitResult {
  targetId: string;           // ID of player hit
  zone: HitboxZone;           // Which body zone was hit
  multiplier: number;         // Damage multiplier for that zone
  distance: number;           // Distance from shooter to hit point
  hitPosition: Vec3;          // World position of impact
  hitNormal: Vec3;            // Surface normal at impact (for particle direction)
  isHeadshot: boolean;        // Convenience flag
  armorProtected: boolean;    // Whether this zone has armor
}
```

**createHitboxSet(id: string): HitboxSet**
- Creates a new set of 7 hitboxes for a player using the default template
- Each hitbox tagged with the player's ID

**updateHitboxPositions(hitboxSet: HitboxSet, boneWorldPositions: Record<string, Vec3>): void**
- Takes current bone world-space positions (extracted from Three.js skeleton)
- Updates each hitbox's position/endpoints based on which bone it follows
- Head sphere center = Head bone world position
- Chest box center = Chest bone world position
- Stomach box center = Spine bone world position
- Arm capsules: endA = UpperArm bone world pos, endB = Hand bone world pos
- Leg capsules: endA = Thigh bone world pos, endB = Shin bone end world pos

**testHitscan(rayOrigin: Vec3, rayDir: Vec3, targets: { id: string, hitboxes: HitboxSet }[]): HitResult | null**
- Tests ray against ALL hitboxes of ALL targets
- Returns closest hit result, or null if nothing hit
- ARM PENETRATION: When sorting hits, if closest hit is an arm AND the ray also hits chest/stomach of the same target, return the torso hit instead. Arms are "transparent" — they don't block torso hits behind them. Implementation: collect all hits for each target, if any arm hit exists AND any torso hit exists for same target, discard arm hits and use torso hit.
- Skip any target whose id matches the shooter (prevent self-hit)
- Compute hitNormal: for sphere = normalize(hitPoint - center), for capsule = perpendicular from capsule axis, for box = face normal of entry face

**testEnvironmentHit(rayOrigin: Vec3, rayDir: Vec3, maxDistance: number): { position: Vec3, normal: Vec3 } | null**
- Placeholder for wall/ground hit testing (will use Three.js Raycaster against world geometry in integration)
- For now, return null — will be wired in Plan 05

Export: HitResult, createHitboxSet, updateHitboxPositions, testHitscan, testEnvironmentHit
