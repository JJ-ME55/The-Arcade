**Import types from weapons.ts and combat.ts:** WeaponType, WeaponConfig, HitResult, HitboxZone

**PlayerHealth class:**
```typescript
class PlayerHealth {
  hp: number = 100;
  armor: number = 100;
  hasHelmet: boolean = true;
  alive: boolean = true;

  // Tagging state
  tagSpeedMultiplier: number = 1.0;  // 1.0 = no slowdown
  tagTimeRemaining: number = 0;       // Seconds of tag remaining

  reset(): void  // Full reset: 100 HP, 100 armor, helmet, alive
  update(dt: number): void  // Decay tagging timer
}
```

**DamageResult type:**
```typescript
interface DamageResult {
  damageDealt: number;        // Actual HP lost
  armorDamaged: number;       // Armor points consumed
  killed: boolean;            // Did this shot kill the target?
  isHeadshot: boolean;        // Was it a headshot?
  zone: HitboxZone;           // Which zone was hit
  remainingHp: number;        // Target's HP after damage
  remainingArmor: number;     // Target's armor after damage
  tagSpeedMultiplier: number; // Speed multiplier to apply (for tagging)
}
```

**KillEvent type:**
```typescript
interface KillEvent {
  killerId: string;
  victimId: string;
  weaponType: WeaponType;
  isHeadshot: boolean;
  damageDealt: number;        // Total damage of killing shot
}
```

**DamageSystem class:**
- `players: Map<string, PlayerHealth>` — tracks all player health states
- `registerPlayer(id: string): void` — creates PlayerHealth entry
- `removePlayer(id: string): void`
- `getHealth(id: string): PlayerHealth | undefined`
- `resetPlayer(id: string): void` — calls PlayerHealth.reset()
- `resetAll(): void` — resets all players (round start)

- `applyDamage(shooterId: string, hitResult: HitResult, weaponConfig: WeaponConfig): DamageResult | null`

  CS:S damage calculation order (CRITICAL — from RESEARCH.md Pitfall 4):
  1. `rawDamage = weaponConfig.baseDamage * hitResult.multiplier`
  2. Armor reduction check:
     - Head zone: only reduced if target.hasHelmet AND target.armor > 0
     - Chest/Stomach/Arms: reduced if target.armor > 0
     - Legs: NEVER reduced by armor
  3. If armor protects this zone: `armorAbsorption = rawDamage * 0.5` (armor absorbs 50%)
     - `damageDealt = rawDamage - armorAbsorption`
     - `armorConsumed = armorAbsorption * 0.5` (armor durability: takes half of what it absorbs)
     - Clamp armorConsumed to remaining armor
     - If armor runs out mid-calculation, remaining damage goes to HP at full rate
  4. `target.hp -= damageDealt`
  5. `target.armor -= armorConsumed`
  6. If target.hp <= 0: target.alive = false, generate KillEvent

  **Tagging calculation (COMBAT-08):**
  - On any hit: `tagDuration = 0.1` seconds (100ms, CS:GO style)
  - Speed reduction scales with damage: `tagMultiplier = 1.0 - Math.min(0.8, damageDealt / 100)`
    - 10 damage = 10% slow, 50 damage = 50% slow, 80+ damage = capped at 80% slow
  - New hit replaces existing tag (does not stack)
  - Update: `tagTimeRemaining -= dt`. When <= 0, tagSpeedMultiplier returns to 1.0

- `applyKnifeDamage(attackerId: string, targetId: string, isBackstab: boolean): DamageResult | null`
  - Left-click: 40 damage, ignores armor completely
  - Right-click backstab: 200 damage (instant kill through any HP/armor)
  - Backstab detection: compare attacker's forward direction dot product with target's forward direction. If dot < 0 (attacking from behind within ~90 degree cone), it's a backstab. The actual dot product check will happen in integration (needs 3D positions), so this method takes isBackstab as a parameter.
  - Still applies tagging on non-lethal hits

- `update(dt: number): void` — called each tick, updates tagging timers for all players

**Damage verification values (from CONTEXT.md, must match exactly):**

- Knife left: 40 damage flat. Knife right backstab: 200 damage (instant kill).
