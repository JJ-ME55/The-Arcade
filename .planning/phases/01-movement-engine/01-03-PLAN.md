  <files>

  <action>
    Build the complete settings infrastructure.

    **KeyBindingManager.ts:**
    - Manages action-to-keycode mappings (see 01-RESEARCH.md Key Binding System example)
    - Default bindings from 01-CONTEXT.md:
      - forward: 'KeyW', back: 'KeyS', left: 'KeyA', right: 'KeyD'
      - jump: 'Space', crouch: 'ControlLeft', walk: 'ShiftLeft'
      - reload: 'KeyR', weapon1: 'Digit1', weapon2: 'Digit2', weapon3: 'Digit3'
      - shoot: 'Mouse0', aim: 'Mouse2'
    - rebind(action, newKeyCode): returns false if conflict exists (same key bound to different action)
    - getAction(keyCode): returns action name or undefined
    - getBinding(action): returns current keycode for action
    - reset(): restore all defaults
    - Reverse map (keyCode -> action) for fast lookup during input polling
    - localStorage persistence with key 'fps-game-keybindings'
    - Load on construction, save on every rebind

    **SettingsManager.ts:**
    - Centralized settings store for non-keybinding settings:
      - mouseSensitivity: number (default 0.002 radians/pixel, range 0.0005 to 0.01)
      - crosshair: { size: number, thickness: number, gap: number, color: string }
        - Defaults: size=10, thickness=2, gap=4, color='#00ff00'
    - get(key) and set(key, value) methods
    - localStorage persistence with key 'fps-game-settings'
    - Emits 'settings:changed' via EventBus when any setting changes

    **InputManager.ts modifications:**
    - Replace hardcoded key checks with KeyBindingManager lookups
    - pollInput() uses keyBindingManager.getAction(keyCode) to determine which actions are active
    - Constructor accepts KeyBindingManager instance
    - Mouse sensitivity read from SettingsManager (passed as constructor param or referenced)

    **Crosshair.ts:**
    - Pure DOM-based crosshair overlay (see 01-RESEARCH.md Crosshair Implementation example)
    - Four divs positioned at screen center: top, bottom, left, right lines
    - Respects settings: size (line length), thickness (line width), gap (distance from center), color
    - update(settings): applies new crosshair settings immediately by updating div styles
    - Listens for 'settings:changed' events to auto-update
    - Positioned with CSS: position:fixed, top:50%, left:50%, transform:translate(-50%,-50%), pointer-events:none, z-index:1000

    **SettingsMenu.ts:**
    - Full-screen overlay that appears when pressing Escape (or a designated key like Tab/F10)
    - Layout (plain HTML/CSS, no framework):
      - Left panel: Key Bindings list
        - Shows each action name and its current binding (e.g., "Move Forward: W")
        - Clicking a binding puts it in "listening" mode (text changes to "Press a key...")
        - Next keypress/mouseclick binds that key to the action
        - Conflict detection: if key already bound, show warning, don't rebind
        - "Reset Defaults" button at bottom
      - Right panel: Mouse & Crosshair settings
        - Mouse sensitivity: slider (0.0005 to 0.01) with numeric display
        - Crosshair size: slider (1 to 30)
        - Crosshair thickness: slider (1 to 5)
        - Crosshair gap: slider (0 to 20)
        - Crosshair color: color input (HTML native color picker)
        - "Reset Defaults" button
      - Close button (X) or Escape to dismiss
    - When open: pointer lock is released, game is paused (or at least input is not processed by game)
    - When closed: pointer lock re-engages on click
    - Styling: dark semi-transparent background, monospace font, CS:S-inspired dark theme
      - Background: rgba(0, 0, 0, 0.85)
      - Text: #cccccc
      - Accent: #ff9900 (orange, matching dev-texture theme)
      - Inputs/sliders: styled to match theme

    **index.html modifications:**
    - Add container div for settings menu (hidden by default)
    - Add container div for crosshair

    **main.ts modifications:**
    - Create KeyBindingManager, SettingsManager instances
    - Pass KeyBindingManager to InputManager
    - Create Crosshair and wire to SettingsManager
    - Create SettingsMenu and wire to KeyBindingManager + SettingsManager + Crosshair
    - Wire Escape key to toggle settings menu (when pointer locked: unlock and show settings; when settings showing: close settings)
    - Wire Movement sensitivity to SettingsManager.mouseSensitivity

    IMPORTANT: The settings menu must NOT interfere with gameplay. When the menu is open, game input is suppressed. When closed, game resumes exactly as before. Escape toggles between gameplay and settings.
  </action>
  <verify>
    1. Press Escape during gameplay -- settings menu opens, pointer unlocks
    2. Click a key binding (e.g., "Move Forward: W"), press a different key (e.g., "I") -- binding updates to "I", and pressing "I" now moves forward
    3. Drag mouse sensitivity slider -- immediately feel different turn speed in-game after closing menu
    4. Adjust crosshair size/thickness/gap/color -- crosshair updates visually in real-time
    5. Close browser tab, reopen -- all settings preserved (key bindings, sensitivity, crosshair)
    6. Click "Reset Defaults" -- all settings return to defaults
    7. `npx tsc --noEmit` produces zero errors
  </verify>
  <done>
    Full key rebinding system with conflict detection. Mouse sensitivity slider. Crosshair customization (size, thickness, gap, color). Settings menu UI with dark CS:S-inspired theme. All settings persist to localStorage across sessions.

  <action>
    Build the debug tools for physics validation (see 01-CONTEXT.md Debug Overlay and 01-RESEARCH.md Debug Overlay example).

    **StatsOverlay.ts:**
    - Wraps stats.js library
    - Creates Stats instance, sets panel to 0 (FPS)
    - Positions in top-left corner (z-index above game, below settings menu)
    - begin() called at start of render frame
    - end() called at end of render frame
    - toggle() to show/hide (follows debug HUD visibility)

    **DebugHUD.ts:**
    - Custom HTML overlay (monospace green text on semi-transparent black background)
    - Positioned below stats.js panel (top-left, offset down by ~50px)
    - Toggled with F3 key (starts hidden)
    - Displays each frame (updated in render callback):
      - **Velocity:** total speed in u/s (horizontal magnitude, XZ plane) + per-axis (X, Y, Z)
      - **Position:** world coordinates (X, Y, Z) with 2 decimal places
      - **State:** "GROUND" or "AIR"
      - **Height:** "STANDING" or "CROUCHING"
      - **Tick Rate:** actual physics ticks per second (computed from measurement, should be ~64 Hz)
      - **FPS:** rendered frames per second (from stats.js or own counter)
    - Reference values displayed alongside actuals for quick comparison:
      - "Ground speed: 247.5 / 250 expected" (or similar format)
    - Tick rate measurement: count physics ticks in a 1-second window, display as running average

    **main.ts modifications:**
    - Create StatsOverlay and DebugHUD instances
    - Wire StatsOverlay.begin() at start of render, StatsOverlay.end() at end of render
    - Wire DebugHUD.update() in render callback with current player state
    - Wire F3 keypress to toggle both StatsOverlay and DebugHUD visibility
    - Pass tick rate from GameLoop to DebugHUD (GameLoop should track and expose actual tick rate)

    The debug HUD is the primary tool for validating that physics match CS:S. Reference values from 01-CONTEXT.md:
    - Ground run speed (knife): ~250 u/s
    - Counter-strafe to zero: within 1-2 ticks (~15-30ms)
    - Bhop speed gain: measurable increase per chained hop with air strafe
    - Crouch height reduction: ~45%

    If the actual values don't match expected, the physics constants in Movement.ts should be tuned during the checkpoint in Task 3.
  </action>

    5. Counter-strafe (W then S) -- velocity drops to 0 within 15-30ms visible in overlay
    6. Bunny hop + air strafe -- velocity shows increasing values above 250 u/s
    7. Jump -- state changes from "GROUND" to "AIR" and back
    8. Crouch -- height shows "CROUCHING"
    9. Tick rate shows ~64 Hz (within 63-65 Hz)
    10. stats.js panel shows 60+ FPS during all movement
    11. `npx tsc --noEmit` produces zero errors
  </verify>
