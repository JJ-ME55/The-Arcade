  <action>
    Build the core engine runtime. Each file is a focused module:

    **EventBus.ts:**
    - Typed publish/subscribe system using Map of event name to Set of callbacks
    - Methods: on(event, callback), off(event, callback), emit(event, data)
    - Used for decoupled communication between engine subsystems (e.g., 'pointerlock:locked', 'pointerlock:unlocked')

    **GameLoop.ts:**
    - Fixed-timestep accumulator pattern (see 01-RESEARCH.md Pattern 1)
    - FIXED_DT = 1/64 (15.625ms, 64 ticks per second)
    - MAX_FRAME_TIME cap at 0.25s to prevent spiral of death
    - Uses requestAnimationFrame for render loop
    - Each frame: accumulate time, run physics steps in while loop, interpolate for render
    - Constructor takes callbacks: onPhysicsStep(dt: number), onRender(alpha: number)
    - Tracks actual tick rate for debug display
    - start() and stop() methods

    **InputManager.ts:**
    - Event-driven capture: keydown/keyup listeners set Map<string, boolean> state using event.code
    - Mouse: accumulate movementX/movementY deltas on mousemove (only when pointer locked)
    - pollInput() returns InputSnapshot: { forward, back, left, right, jump, crouch, walk, mouseDelta: {dx, dy} }
    - Polls use default bindings initially: KeyW, KeyS, KeyA, KeyD, Space, ControlLeft, ShiftLeft
    - Mouse delta is consumed (reset to 0) after each poll
    - Blocks browser defaults with event.preventDefault() for game keys (Space scroll, Ctrl+W close, F-keys)
    - Does NOT handle key rebinding yet (Plan 03)

    **Renderer.ts:**
    - Creates THREE.WebGLRenderer targeting the #game-canvas element
    - Sets pixel ratio to window.devicePixelRatio (capped at 2)
    - Sets size to window.innerWidth x window.innerHeight
    - Handles window resize events
    - Creates THREE.Scene with a solid background color (#87CEEB sky blue or similar)
    - render(camera) method called each frame
    - Exposes scene for adding objects

    **Camera.ts:**
    - Creates THREE.PerspectiveCamera (FOV 90, near 0.1, far 1000)
    - Wraps PointerLockControls from 'three/examples/jsm/controls/PointerLockControls'
    - Requests pointer lock on canvas click with { unadjustedMovement: true }, falls back to standard if NotSupportedError
    - Emits 'pointerlock:locked' and 'pointerlock:unlocked' events via EventBus
    - Exposes getObject() for position (used by physics), getDirection() for forward vector
    - Camera Y position will be controlled by physics (Plan 02), for now set to 1.7 (eye height)

    **TestArena.ts:**
    - Creates the orange/yellow dev-texture block arena for movement validation (see 01-CONTEXT.md)
    - Uses MeshStandardMaterial with flat orange/yellow colors (no textures)
    - Arena layout:
      - Large flat ground plane (100x100 units) - MeshStandardMaterial color #D4A853 (sandy orange)
      - 4 elevated platforms at heights 2, 4, 6, 8 units (each ~10x10 units) - color #C4933D
      - 3 ramps connecting ground to platforms at ~30-degree angles - color #B8862B
      - 1 long straight corridor (50 units long, 4 units wide, 3 units tall walls) - color #DAA520
      - Several box obstacles scattered on ground (various sizes 1-3 units) - color #CD950C
    - Ambient light (intensity 0.6) + directional light (intensity 0.8, from above-front)
    - Returns array of THREE.Box3 representing all solid geometry for collision detection (exported as getCollisionBoxes())
    - All meshes added to the scene

    **main.ts:**
    - Entry point that wires everything together:
      1. Create EventBus
      2. Create Renderer (targets #game-canvas)
      3. Create Camera (attaches PointerLockControls to canvas)
      4. Create TestArena (adds geometry to scene)
      5. Create InputManager
      6. Create GameLoop with:
         - onPhysicsStep: polls input (unused for now, physics in Plan 02)
         - onRender: calls renderer.render(camera)
      7. Start game loop
    - Show/hide "Click to Play" overlay based on pointer lock state
    - Player position is static for now (no physics) -- camera stays at (0, 1.7, 0) and only rotates via mouse

    Use Three.js Vector3 for all position/direction math. Do NOT use any physics library.
    Import PointerLockControls from 'three/examples/jsm/controls/PointerLockControls.js' (note the .js extension for Vite compatibility).
  </action>
