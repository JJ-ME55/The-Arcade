  <action>
Create a Python script `visual/export_glb.py` that:

1. Opens `visual/arena_map.blend`
2. Iterates all materials in the file and replaces their node trees with simple Principled BSDF:
   - Material named "W" or containing "W" -> Base Color: light grey (#AAAAAA / 0.667, 0.667, 0.667)
   - Material named "O" or containing "O" (orange) -> Base Color: orange (#DD7722 / 0.867, 0.467, 0.133)
   - Material named "DarkGrey" or containing "Dark" -> Base Color: dark grey (#555555 / 0.333, 0.333, 0.333)
   - Any other material -> Base Color: medium grey (#888888)
   - Set Roughness to 0.9 for all materials (matte look, no reflections)
   - Set Metallic to 0.0 for all materials
   - Clear all existing nodes before adding the Principled BSDF + Output nodes
3. Creates two Empty objects as spawn point markers:
   - "spawn_red" at position (0, -23, 0) with custom property `spawnTeam` = "red"
   - "spawn_blue" at position (0, 28, 0) with custom property `spawnTeam` = "blue"
   - Note: In Blender, Y is the depth axis. When exported to GLTF, Blender Y becomes Three.js -Z and Blender Z becomes Three.js Y (GLTF uses Y-up convention, Blender uses Z-up). The GLTF exporter handles this automatically.
4. Exports to `visual/arena_map.glb` with these settings:
   - Format: GLB (binary)
   - Include: Custom Properties enabled
   - Transform: +Y Up (standard GLTF convention, exporter converts from Blender Z-up)
   - Geometry: Apply Modifiers enabled
   - No animations, no skinning

Then run the script via Blender CLI:
```
"C:\Program Files\Blender Foundation\Blender 4.4\blender.exe" --background --python visual/export_glb.py
```

IMPORTANT: The script must use `bpy.ops.wm.open_mainfile()` to open the .blend file first, OR be invoked with the blend file as argument:
```
"C:\Program Files\Blender Foundation\Blender 4.4\blender.exe" --background visual/arena_map.blend --python visual/export_glb.py
```

The second form is cleaner -- Blender opens the .blend file, then runs the script.

After running, verify the GLB file exists and has reasonable size (should be 100KB-2MB for 153 mesh objects with flat materials, no textures).
