What's locked (do not let the designer / AI change these)

  • Cash readout — bold yellow-gold numerals, e.g. $12,450 (we overlay this as real text, never baked into the image)
  • The green screen — dark-green CRT glass, green terminal text, faint scanlines + glow
  • Shell — weathered olive-grey steel, rust + grime + rivets, thick rounded polished-silver frame
  • Red round X close button, top-right
  • Chrome italic machine names ("Mineral Processor 3000" etc.) — typeset by us, not painted by AI

The golden rule: generate the blank machine shell only — no text, no numbers, no logos. AI mangles type. All copy (logos, prices, cash, CRT text) and the ore/drill renders get overlaid in-engine, which also keeps them crisp and localisable.

How to use the results

  • Generate wide (1792×1024), 3–4 variations per machine; pick the best shell.
  • Reuse the preamble verbatim across all three so they read as one family.
  • If a result comes back too clean: append "heavier rust, more grime and scratches". If it invents text: regenerate and restate "no text of any kind".
  • We then composite on top, in-engine/HTML: chrome italic logos, the yellow cash, all green CRT text, our faceted ore icons and drill renders, prices, LEDs' on/off state. That keeps every word crisp and editable.
  • Same approach works for a human designer: hand them this pack + the reference screenshots + our current HTML screens as the layout spec.
